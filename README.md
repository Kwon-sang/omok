# 오목 게임

> CMD 콘솔 기반 오목 게임

<br><br>

## 1. 요구사항 설계
- 게임 시작 시, Player1 과 Player2 의 이름을 설정 가능하다.
- 바둑판의 크기는 오목 정식 규격인 15*15이며, 돌을 둘 수 있는 위치 또한 이와 같다.
- 바둑판에서 각 행과 열의 위치를 알 수 있도록, 출력된 바둑판에는 **행과 열의 인덱스**를 포함해야 한다.
- 사용자의 입력은 정해진 **입력 포맷**을 가지며, 잘못된 입력시에는 에러 메시지와 함께 **재입력**을 요구한다.
   * 바둑돌을 놓는 위치에 대한 입력은 공백을 기준으로 '행인덱스 열인덱스'의 형태이며 각 인덱스 번호는 숫자만 가능하다. ex)'3 4'
   * 바둑판의 크기를 벗어나는 위치에 돌을 둘 수 없으며, 이미 돌을 둔 자리 또한 돌을 둘 수 없다.
   * 바둑돌을 놓는 위치에 대한 입력에서 Q 혹은 q를 입력시, 즉시 게임을 종료할 수 있다.
- 각 플레이어가 돌을 놓은 후, 화면에 바둑판의 상태를 출력해야 한다.
- 각 플레이어는 번갈아가며 흑돌과 백돌을 두어야 한다.
- 오목을 검증하는 로직을 구성해야 하며, 플레이어의 오목 달성 시 플레이어의 이름과 함께 게임 승리 문구가 출력되어야 한다.

<br><br>

## 2. 구조 설계
 - 사용자 입력 메시지, 에러메시지 등 문자열은 하드코딩이 아닌, `settings.py`에서 변수로 관리.
 - 사용자 입력을 위한 함수들은 `commands.py` 모듈에서 구성.
 - 바둑판의 생성, 돌 위치 삽입, 바둑판 상태 출력 등의 바둑판 전반에 대한 로직은 `board.py`의 `Board` 클래스로 관리.
 - `main.py`에서 `run()` 메소드에서 게임 실행 로직을 구성
 - 각 모듈의 구성에 있어서 낮은 의존성과 높은 응집도를 고려.

<br><br>

## 3. 모듈 구조

* `main.py` : `run()`메서드에서 게임 실행 로직을 구성
* modules
    * `settings.py` : 바둑판 마킹, 입력 메시지, 오류 메시지 등 *텍스트 변수* 를 관리.
    * `commands.py` : *사용자 입력* 관련한 메서드 정의하며, 입력 Format을 검증하여 반환.
    * `boards.py` : `Board` 클래스를 통해 바둑판, 포석 위치값, 바둑판 생성 및 출력 등 바둑판과 관련한 로직을 응집.

<br><br>

## 4. 고민한 사항들
- 각 입출력 메시지와 에러 메시지는 호출한 함수에 하드코딩해야 하는가?
  * 이 경우 변경에 취약한 구조이며, 안티 패턴
  * 입출력 문자열, 에러메시지 문자열, 바둑판 마킹 문자 등의 문자 텍스트 변수를 따로 관리하는 파일을 구성하여 해결.
- 낮은 의존성과 높은 응집도를 어떻게 달성할 것인가?
- 사용자 입력에 대하여 검증 로직은 과연 사용자 입력을 모듈에서 구성해야 하는가?
  * 이 경우 바둑판에 돌을 놓을 위치에 대한 사용자 입력과 입력값 검증 로직에서 '바둑판'에 대한 의존성이 발생
  * 사용자 입력 모듈은 입력 포맷에 대해 검증을 수행하고, 바둑판 정보가 필요한 검증사항은 바둑판 모듈에 위임하는 것이 맞다고 판단.
  * 바둑판에 대한 정보는 바둑판 모듈에서 구성되므로, 이에 의존성을 갖는 사항들을 구성함으로써 높은 응집도를 가질 수 있을 것으로 판단.
